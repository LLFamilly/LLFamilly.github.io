三、浏览器工作原理

浏览器的组成

  • 人机交互部分（UI）
  • 网络请求部分（Socket）
  • JavaScript引擎部分（解析执行JavaScript）
  • 渲染引擎部分（渲染HTML、CSS等）
  • 数据存储部分（cookie、HTML5提出的LocalStorage、SessionStorage）

主流渲染引擎介绍

1、渲染引擎 又叫 排版引擎 或 浏览器内核, 主流的 渲染引擎 有：

  • Chrome: Blink引擎（WebKit的一个分支）。
  • Safari: WebKit引擎，windows版本2008年3月18日推出正式版，但苹果已于2012年7月25日停止开发Windows版的Safari。
  • FireFox: Gecko引擎。
  • Opera: Blink引擎(早期版使用Presto引擎）。
  • Internet Explorer: Trident引擎。
  • Microsoft Edge: EdgeHTML引擎（Trident的一个分支）。

2、主流渲染引擎工作原理

  1. 解析HTML构建Dom树（Document Object Model，文档对象模型），是W3C组织推荐的处理可扩展置标语言的标准编程接口。
  2. 构建渲染树，渲染树并不等同于Dom树，因为像head 或 display: none这样的元素就没有必要放到渲染树中了，但是它们在Dom树中。
  3. 对渲染树进行布局，定位坐标和大小、确定是否换行、确定position、overflow、z-index等等，这个过程叫"layout" 或 "reflow"。
  4. 绘制渲染树，调用操作系统底层API进行绘图操作。

浏览器访问网站过程

  1. 在浏览器地址栏中输入网址。

    1. 浏览器通过用户在地址栏中输入的URL构建HTTP请求报文。

    2. 浏览器发起DNS解析请求，将域名转换为IP地址。

      1. 浏览器将请求报文发送给服务器。
      2. 服务器接收请求报文，并解析。

      3. 服务器处理用户请求，并将处理结果封装成HTTP响应报文。

      4. 服务器将HTTP响应报文发送给浏览器。

      5. 浏览器接收服务器响应的HTTP报文，并解析。
      6. 浏览器解析HTML页面并展示，在解析HTML页面时遇到新的资源需要再次发起请求。
      7. 最终浏览器展示出了页面

# 四、Web开发本质
1. 请求，客户端发起请求。
2. 处理，服务器处理请求。
3. 响应，服务器将处理结果发送给客户端
4. 服务器响应完毕后，客户端继续处理：
   - 浏览器：解析服务器返回的数据
      - iOS、Android 客户端，解析服务器返回的数据，并且通过iOS或Android的UI技术实现界面的展示功能

  # 五、Node.js介绍
  ## 1、什么是Node.js?
  - Node.js是一个开发平台，就像Java开发平台、.Net开发平台、PHP开发平台、Apple开发平台一样`开发平台：有对应的编程语言、有语言运行时、有能实现特定功能的API（SDK：Software Development Kit）`
  - 在这个平台开发所使用的语言是JavaScript语言
  - 在这个开发平台下可以开发控制台程序（CLI）、桌面程序GUI（参见atom编辑器）、Web程序
  - Node.js设计主要是为了开发网络应用程序
  - Node.js底层是通过C/C++语言实现的
  - Node.js内置部分API，通过NPM可以使用更多第三方功能模块
